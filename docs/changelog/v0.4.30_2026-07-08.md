# Changelog v0.4.30 - 2026-07-08

## 修复 (Bug Fixes)

### 修复删除工作区时容器资源清理失败

#### 问题描述

- 现象：删除工作区时，`backend-spawn.log` 出现 `AttributeError: 'WorkspaceRegistryService' object has no attribute 'workspace_root'`，导致容器资源清理逻辑被异常跳过。删除接口虽返回 200，但清理步骤未实际执行，可能与删除后页面状态异常相关。
- 原因：`WorkspaceRegistryService._cleanup_workspace_containers` 中错误地访问了不存在的 `self.workspace_root` 属性。该类实际保存工作区根目录的属性为 `self.base_dir`。

#### 改动内容

- 将 `ContainerResourceService` 初始化参数从 `self.workspace_root` 修正为 `self.base_dir`，使删除工作区时能正确构造容器资源服务并执行容器清理。

#### 文件变更

- `apps/backend/app/services/workspace_registry.py`: 修正 `_cleanup_workspace_containers` 中 `ContainerResourceService` 的初始化参数。

### 修复 `fnm install lts` 安装后状态显示异常

#### 问题描述

- 现象：通过 `fnm install lts` 安装 Node.js 后，运行环境面板显示环境状态为 `registered`（已登记）而非 `ready`（可使用）。前端看到"Node 环境已登记但不能使用"，影响创建工作区或手动安装 Node 环境时的体验。
- 原因：`install_node_version` 安装完成后用 `fnm list` 校验时，直接检查输入字符串 `"lts"` 是否在输出中。但 `fnm list` 输出的是解析后的具体版本号（如 `v22.15.0`），导致校验失败。

#### 改动内容

- 新增 `_parse_fnm_list_versions` 工具函数，从 `fnm list` 输出中解析已安装的具体版本号。
- 安装完成后，对 `lts`/`lts-iron` 等别名解析为实际安装的版本号；对具体版本号则按前缀匹配已安装版本。
- 将环境记录中的 `node_version` 和 `display_name` 更新为实际版本号，避免注册表中保留别名字符串。

#### 文件变更

- `apps/backend/app/services/node_runtime.py`: 新增 `_parse_fnm_list_versions` 并修正 `install_node_version` 的版本校验逻辑。
- `apps/backend/tests/test_node_runtime.py`: 新增 `_parse_fnm_list_versions` 的单元测试。

## 测试 (Tests)

- `delete_workspace` / `workspace_delete` / `cleanup_workspace` / `runtime_env` 相关回归测试 28 个全部通过。
