# 上下文压缩

复杂任务中 Agent 的工具调用和中间结果会迅速膨胀上下文，接近模型上下文窗口上限。上下文压缩在保护关键信息的前提下削减上下文体积。

## 两层压缩

### 第一层：无损截断（Snip）

对过长的工具输出进行截断：

- 保留前 50% 和末尾 25% 的内容
- 中间部分标记为已截断，保留行数和字节数统计
- 不改变保留内容的格式和语义

这一层是纯规则截断，不涉及 LLM，速度快且确定性高。

### 第二层：LLM 结构化摘要

调用 LLM 将消息列表压缩为结构化 XML。摘要按 6 个优先级组织信息：

1. 当前焦点：正在处理的任务和当前状态
2. 已完成任务：已完成的步骤和结果
3. 活跃问题：未解决的错误和阻塞
4. 代码状态：修改了哪些文件、当前代码的关键状态
5. 环境信息：工作区路径、运行中的进程等
6. 其他上下文

压缩后的 XML 摘要体积远小于原始消息列表，但保留了继续推进任务所需的核心信息。

## 保护机制

**系统提示词不压缩**：Agent 行为约束相关的系统提示词始终完整保留，不受压缩影响。

**工具调用对保护**：tool_call 和 tool_result 作为边界保留，不会被拆分到不同压缩块中。压缩粒度是消息边界，不是 token 边界。

**多模态内容剥离**：压缩后的消息中移除图片 base64 数据，只保留文本内容和文件引用。压缩后的 Agent 不再能直接"看"到之前的图片，但保留了文件路径可以重新读取。

## 优雅降级

LLM 摘要如果失败（最多重试 3 次，指数退避），返回原始消息列表，不丢失任何状态。压缩失败不会导致任务中断。

## 手动压缩

用户可以从对话栏头部手动触发上下文压缩。手动压缩通常用于以下场景：

- 对话历史很长，但还想在当前会话继续
- TokenUsageBar 显示上下文占用接近窗口上限
- 切换话题前清理旧上下文

## 上下文占用显示

TokenUsageBar 在对话栏头部显示：

- 当前上下文占用 token 数
- 模型上下文窗口大小
- 占用比例

这是当前轮提示词大约占用的窗口比例，不是会话累计消耗。
